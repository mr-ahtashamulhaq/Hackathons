# Contributing to Transit Accessibility App

Thank you for contributing! Please follow these rules:

## Branching
- All work must be done in a separate branch.
- Branch names: feature/feature-name or bugfix/bug-name.

## Pull Requests
- Open a PR when your work is ready to merge.
- Include a clear description of your changes.
- Wait for review and approval before merging.

## Code Guidelines
- Keep code clean and well-commented.
- Follow the project folder structure.

## Documentation
- Update README.md or docs/ if you add new features.
- Accessibility improvements should always be documented in docs/accessibility.md.
